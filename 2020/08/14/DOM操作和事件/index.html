DOM操作和事件

发表于 2020-08-14 更新于 2020-08-15 分类于 笔记 ， JavaScript学习笔记 阅读次数： Valine：
本文字数： 3.2k 阅读时长 ≈ 3 分钟

DOM

DOM映射

我们平时写 HTML 的时候，大多都认为 HTML 是平面的东西，一堆文字包裹在一堆标签中间。但实际上像 HTML 和 XML 这种形式的文档都是树状结构， 也对应数据结构中的树。
例如： 

<html>
  <head>
    <title>youkeda</title>
  </head>
  <body>
    <div>
      <h1>优课达</h1>
      <p>学的比别人好一点</p>
    </div>
  </body>
</html>

DOM访问

选择器查询：querySelector()

document.querySelector('main .core .subtitle');

迭代查询
当我们得到 subtitle 元素后，我们还可以利用这个元素，继续筛选器内部元素，比如我们想筛选器内部的 a 标签，我们可以继续完善代码： 

let subtitle = document.querySelector('main .core .subtitle');
console.log(subtitle.querySelector('a'));

选择器全量查询：querySelectorAll()

例子： 

document.querySelectorAll('input');

Dom属性

类别（元素节点、特性节点、文本节点……）

1.标签：元素节点（nodeName()获取标签名）
2.标签属性：（一个一节点）特性节点（nodeName()获得属性key，nodeValue()获得属性Value）
3.纯文本：文本节点（nodevalue()获得文本内容）
注：attributes可以获取元素节点的所有属性，得到的结果是一个字典，通过属性 Key 获取对应的特性节点

DOM内容

1.outerHTML //整个DOM节点的HTML代码
2.innerHTML //DOM节点 内部 HTML 代码
3.innerText //DOM 内部纯文本内容

DOM亲属

1.firstChild //指定节点的第一个子节点
2.lastChild //指定节点的最后一个子节点
3.childNodes //指定节点的子节点的集合
4.parentNode //指定节点在 DOM 树中的父节点

DOM样式

1.classList //classList 数组方式存储所有的 class 名称
2.style //对象或字典的方法存储 CSSStyle

DOM数据属性（dataset获取）

对文章而言，除了文章内容，我们还有其他额外数据，例如：段落，字数, 分类，etc… 

<article data-parts="3" data-words="1314" data-category="python">
    ...
 </article>

注意，数据属性很重要，在前端中广泛应用，大家有兴趣可以用 chrome 开发者工具去试试看看一下其他网站的代码。
获取数据代码： 

const article = document.querySelector('article');
console.log(article.dataset);

DOM操作

创建标签节点

1.const div = document.createElement('div');//创建div标签
2.const txt = document.createTextNode('优课达-学的比别人好一点');//创建文本

添加新节点

1.appendChild(newNode)//在所以儿子节点之后添加
2.inserBefore(newNode,referenceNode)//在某个目标儿子节点之前添加

设置样式、属性

1.img.setAttribute('style', 'width: 100%; height: 100%;');//和在HYML中写style语法一样（全换）
2.dom.style.color = 'xxxx';//单独设置某些属性
注：setAttribute()还可以设置其他HTML属性，如：id、src等

innerHTML

1.innerHTML=’’//清空select节点所以后代内容
2.innerHTML=’内容’//设置select节点内部代码

DOM事件

DOM绑定事件

1.dom.onclick=function(){}//只能在该dom节点上绑定一个事件
2.addEventListener('click',function(){})//可以绑定多个事件
注：推荐使用addEventListener()

表单元素事件

1.focus: 表单组件（Input, Textarea, etc..）获取焦点事件
2.blur: 表单组件（Input, Textarea, etc..）失去焦点事件
3.input: 只要输入值变化就会触发
4.change: 输入框失去焦点才会触发

鼠标事件

1.click: 点击事件
2.dblclick: 双击事件
3.mousedown: 在元素上按下任意鼠标按钮。
4.mouseenter: 指针移到有事件监听的元素内。
5.mouseleave: 指针移出元素范围外（不冒泡）。
6.mousemove: 指针在元素内移动时持续触发。
7.mouseover: 指针移到有事件监听的元素或者它的子元素内。
8.mouseout: 指针移出元素，或者移到它的子元素上。
9.mouseup: 在元素上释放任意鼠标按键。
注：移动主要用mouseover和mouseout

键盘事件

1.keydown: 键盘按下事件
2.keyup: 键盘释放事件

视图事件

1.scroll: 文档滚动事件
2.resize: 窗口放缩事件

资源事件

load: 资源加载成功事件

事件的冒泡、捕获、委托

冒泡

触发孩子事件，使得父亲事件也触发（一直到根节点）
例如：父亲和孩子都绑定了点击事件，点击孩子会先触发孩子点击事件，接着就触发父亲点击事件，可能会导致一些问题。所以要阻止冒泡

阻止冒泡：e.stopPropagation() 

// ......省略
likeBtn.addEventListener('click', function(e) {
  // 点击事件
  e.stopPropagation()

// ......省略

捕获

捕获和冒泡是完全相反的，冒泡是从当前元素沿着祖先节点往上冒泡，而捕获是从根 HTML 节点开始依次移动到当前元素。
如果想在捕获阶段监听事件，我们需要传递第三个参数为true， 代码如下： 

dom.addEventListener('click', function() {}, true);

委托

委托其实是冒泡事件的一种应用，这个概念依赖于这样一个事实,如果你想要在大量子元素中单击任何一个都可以运行一段代码，您可以将事件监听器设置在其父节点上，并让子节点上发生的事件冒泡到父节点上，而不是每个子节点单独设置事件监听器。
例子： 

const box = document.querySelector('.box');

box.addEventListener('click', function(e) {
  // 注意box区域比img大，如果点击在空白间隔区域，那么返回的节点将不会是IMG，需要特殊处理一下
  if (e.target.nodeName === 'IMG') {
    document.body.style.backgroundImage = `url(${e.target.src})`;
  }
});

注：target 表示真实响应事件的 DOM 节点
